CopyResource Annotation

The CopyResource annotation can be used to apply a new theme to your specifications, tweak the existing styling, or add new css, javascript, images, or other resources.

The CopyResource annotation can be applied to all classes in the class hierarchy - each class will be inspected in turn (parent first) for the annotation. Each annotation will be processed in isolation to others so they will operate independently. You will need to ensure that there are no conflicts.

The annotation has the following options:

removeDefaultCSSremove the default concordion css styling
sourceFilesOne or more files to add to the generated specification:
  • File name starting with '/' is relative to the root folder
  • File not starting with '/' is relative to the class the annotation is applied to
  • Supports wild card characters ("/images/*.png", "/css/*.*")
  • If no file is found an exception will be thrown
  • Will inspect the HTML and remove any links from the head section that contain "concordion.css" or any of the CSS or JavaScript files that have been added
  • CSS and JS files are treated a little differently than other files:
    • the specifications html will be updated with the correct link (if linked) or content (if embedded)
    • CSS is appended to the end of the head section so that any styling will override the default Concordion styles
    • any existing links to the resource file will be removed
    All other file types must be added to the specification at design time by whomever is writing the specification.
insertTypeEMBEDDED or LINKED - applies to css and js files only

Single Resource

When the fixture is annotated with @CopyResource(sourceFiles = "demo.txt") and the file demo.txt is located in the same folder as the specification, the resource is copied next to the specification.

Multiple Resources

When the fixture is annotated with @CopyResource(sourceFiles = { "*.css", "/spec/concordion/annotation/demo.txt" }, insertType = InsertType.LINKED) and the files demo.css and demo.txt are located in the same folder as the specification, the resources are copied next to the specification.

Delete existing links

When the fixture is annotated with @CopyResource(sourceFiles = "demo.css", insertType = InsertType.LINKED) and the specification contains:
<link href="../../../concordion.css" rel="stylesheet" type="text/css" /> <link href="demo.css" rel="stylesheet" type="text/css" /> The concordion.css link will be removed and the demo.css will be removed and replaced with a new one.

Parent classes annotations are processed AND default concordion CSS can be removed

When the fixture is annotated with @CopyResource(sourceFiles = "demo.css", insertType = InsertType.LINKED) and the fixtures parent is annotated with @CopyResource(removeDefaultCSS = true) and the specification contains:
<link href="../../../concordion.css" rel="stylesheet" type="text/css" /> <link href="demo.css" rel="stylesheet" type="text/css" /> The concordion.css link will be removed and the demo.css will be removed and replaced with a new one.

File not exists

When the fixture is annotated with @CopyResource(sourceFiles = "im-not-here.txt") an exception is thrown.
